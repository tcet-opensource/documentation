## Navbar Component

The `Navbar` component is responsible for rendering a navigation bar with a logo, menu links, and a responsive hamburger menu for smaller screens.

### Defining Components

1. We defined an array for navigation links (`navlinks`) that will be displayed in the navigation bar. Each link has a title, link URL, and an optional `newtab` flag indicating whether the link should open in a new tab.

2. We defined the `Navbar` functional component along with the initialization of the `open` state variable using the `useState` hook. This variable is used to control the state of the responsive menu.
```jsx
const Navbar = () => {
  const [open, setOpen] = useState(false);
}
```
3. Then defining the `handleMenu` function, which toggles the `open` state when the menu button is clicked.
```jsx
const handleMenu = () => 
    setOpen((prev) => !prev);
    console.log(open);
```

### Rendering the responsive hamburger menu for smaller screens:

What about mobile screens, like smartphones? That's where the hamburger menu comes into play. You know, the icon with three horizontal lines stacked on top of each other. When you tap on the hamburger, it reveals a hidden menu with all the links neatly listed. This way, you can easily navigate the website, even on your phone. The code to carry this out:

```jsx
<div className="-mr-2 flex md:hidden">
  <button
    type="button"
    onClick={handleMenu}
    className="inline-flex items-center justify-center p-2 rounded-lg text-white hover:ring-white focus:outline-none focus:ring-2 focus:rig-offset-2 focus:ring-offset-white focus:ring-white"
  >
    <span className="sr-only">Open main menu</span>
    {open === true ? <FaTimes /> : <FaBars />}
  </button>
</div>
```

### Rendering the mobile navigation menu when the hamburger menu is clicked:

```jsx
{open ? (

        <a

          className="text-center cursor-pointer text-gray-300 hover:bg-gray-700 hover:text-white block px-3 py-2 rounded-md text-base font-medium"

        Get Started

  </div>

```
